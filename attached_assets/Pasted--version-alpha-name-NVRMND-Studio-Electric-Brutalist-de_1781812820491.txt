## Overview

Typography baseline relies on SF Pro Display, Arial, sans-serif for hero headline — massive display type dominating the above-fold viewport.

This system uses a 8px base grid with scale values 4, 8, 16, 24, 32, 48, 64, 96, 128.

**Signature traits:**
- Core token rhythm: Token evidence indicates consistent color, spacing, and radius rhythm across visible UI.

## Colors

The palette uses 8 validated color tokens across 1 theme profile. Semantic roles stay attached to observed usage so generation agents can choose accents without inventing new color meaning.

**Semantic naming:**
- **border-text** maps to `black`: Role "text" is grounded by usage context "Primary text, headings, borders, and most foreground elements across header, main, and footer".
- **action-text** maps to `primary-brand-blue`: Role "text" is grounded by usage context "CTA buttons, nav links, interactive elements, and key accent highlights".
- **surface-background** maps to `off-white-light-grey`: Role "background" is grounded by usage context "Page background and surface fills throughout the site".
- **action-background** maps to `white`: Role "background" is grounded by usage context "Button text, icon fills, and select surface overlays".

## Layout

Responsive system uses 3 breakpoint tier(s): mobile, tablet, desktop.

### Responsive Strategy
- **mobile (<= 991px)**: Constrain layout for small viewports and prioritize vertical stacking.
- **tablet (>= 768px)**: Increase spacing and column structure for medium-width viewports.
- **desktop (>= 1024px)**: Expand layout density and horizontal composition for wide viewports.

2. Map spacing and radius directly from token tables before visual polish.
3. Apply component patterns one section at a time and compare against source intent.
4. Keep elevation claims tied to explicit evidence in output.
5. Iterate with smallest diffs and re-check section hierarchy after each change.
